Section 2: Folder Structure & Naming Conventions

Before you write a single line of code, set yourself (and ChatGPT) up for success with a clean folder structure. Trust us — future you will thank you.

The StackPilot Structure

StackPilot uses a modular structure that separates layout, content, and components. This keeps everything organized, scalable, and easy to work with — especially when working alongside ChatGPT. 

stackpilot/
├── index.html               # The master layout
├── content/                 # Modular content files
│   └── index-content.html   # Section 1 (Welcome)
├── assets/                  # Shared layout parts
│   ├── sidebar.html
│   ├── header.html
│   └── footer.html
└── .nojekyll                # Ensures GitHub Pages serves folders correctly
Why this matters: ChatGPT can more reliably generate code and content when it knows exactly where things go. And it makes debugging way easier.

Naming Rules

Use kebab-case for everything (lowercase, no spaces, hyphens instead of underscores). This helps avoid bugs on case-sensitive servers and keeps naming readable for humans and machines alike.

  • section2-content.html — Good
  • Section2_Content.html — Bad
  • MainPage.html — Nope

Bonus: Scaffold Prompt

Create a folder structure for a modular website that includes:

- a content/ folder with modular sections
- an assets/ folder for sidebar, header, and footer
