Section 3: Layout & Architecture

A solid layout is the backbone of every great website — and StackPilot's modular layout makes your site easy to build, maintain, and scale.

One Layout, Many Views

StackPilot uses a single layout file: index.html. It holds the core structure — header, sidebar, footer, and the main content area. Instead of loading a new page every time, we just replace the main content.

Benefits: Faster navigation, cleaner code, and easier updates — especially when working with ChatGPT.

Layout Structure Overview

<body>
  <aside id="sidebar-placeholder"></aside>
  <header id="header-placeholder"></header>
  <main id="main"></main>
  <footer id="footer-placeholder"></footer>
</body>

Each of these placeholders is filled with content using JavaScript and a simple fetch(). That means shared elements like the header or footer only need to be written once.

Content Injection in Action

When you visit #section2 in the URL, the JavaScript loads content/section2-content.html and injects it into the <main> element. 

loadComponent("main", "content/section2-content.html");

Why This Matters

This approach keeps your layout consistent while allowing you to swap in new content, pages, or sections without duplicating the entire HTML structure.

Pro Tip: Think of index.html as your command center — everything else just plugs into it.
